# /issue-report

현재 GitHub 리포지토리의 오픈 이슈를 조회하고, 중요도별로 분류하여 한국어로 보고합니다.

## 실행 단계

1. 오픈 이슈 전체 조회:
   ```bash
   gh issue list --state open --limit 100 --json number,title,labels,createdAt,author,assignees,comments,milestone
   ```

2. 이슈를 아래 기준으로 **높음 / 중간 / 낮음** 3단계로 분류:

   ### 높음 (High)
   - `bug` 라벨이 붙은 이슈
   - 런타임 크래시, 데이터 손실, 보안 관련 이슈
   - 여러 사용자가 동일 문제를 보고한 경우

   ### 중간 (Medium)
   - 수정이 진행 중이거나 거의 완료된 이슈
   - 특정 환경/조건에서만 발생하는 문제
   - 하위 이슈(sub-issue)로 분리된 작업 항목

   ### 낮음 (Low)
   - `enhancement` 라벨 (기능 요청)
   - `question` 라벨이면서 아직 분석 전인 이슈
   - 일정이 미정인 개선 요청

3. 아래 형식으로 한국어 보고서를 출력:

   ```
   ## 오픈 이슈 목록 및 중요도 보고

   현재 **N개**의 오픈 이슈가 있습니다.

   ---

   ### 높음 (High Priority)

   | # | 제목 | 라벨 | 담당자 | 상태 |
   |---|---|---|---|---|
   | #번호 | 제목 | 라벨 | @담당자 | 상태 요약 |

   **사유**: (왜 높음인지 설명)

   ---

   ### 중간 (Medium Priority)
   (동일 형식)

   ---

   ### 낮음 (Low Priority)
   (동일 형식)

   ---

   ### 요약

   | 우선순위 | 건수 | 핵심 액션 |
   |---|---|---|
   | 높음 | N | 주요 조치 사항 |
   | 중간 | N | 주요 조치 사항 |
   | 낮음 | N | 주요 조치 사항 |

   (가장 시급한 작업에 대한 권고 포함)
   ```

4. 각 이슈의 댓글(comments)을 분석하여 현재 진행 상태를 파악하고 상태 컬럼에 반영

## 참고

- 반드시 `gh` CLI를 사용하여 이슈를 조회할 것
- git 리포지토리가 아닌 경우 에러 메시지를 안내할 것
- 이슈 본문과 댓글의 내용을 기반으로 실제 진행 상태를 판단할 것
- 담당자가 없는 이슈는 "미지정"으로 표시
- 보고서는 항상 한국어로 작성
